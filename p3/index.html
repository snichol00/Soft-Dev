Scala

Overview

Scala stands for "scalable language," meaning that it can be scaled up or down to meet the user's needs, using a combination of object-oriented and functional programming. Object-oriented programming will be familiar to java users, and includes the four key tenets of polymorphism, encapsulation, inheritance, and abstraction. In many ways, Scala is a more concise and intuitive version of Java, but its use of functional programming (which will be review below) makes it a substantially different language.

Improvements on Java

  • Scalable: Better support for backend operations
  • Concise: Java tends to be repetitive and demands that the user manually write a lot of functions that are basic and widely used, while scala is a smarter and less verbose language
  • Readability: Without extra clutter, code tends to be more easily digestible and the functions seem more intuitive, allowing easier debugging

Key Features

  • Object-oriented: Modularity makes it easier to troubleshoot
  • Statically typed: Catches errors at compile time, before execution
  • Lazy computation: Only computes what is necessary, which makes it run faster
  • Type safe: doesnâ€™t allow the use of incompatible data types

Functional Programming

        Functional programming is a way of computing that focuses less on defining a function step-by-step and more on the outcome, creating more concise but less explicit code. It relies on the
        machine to fill in the steps in between.

Key Features

  • Declarative rather than imperative: meaning it's more focused on the what is accomplished than how it's done
  • Pure functions: Always the same output for every input
  • Avoids shared state: Shared state is any variables or objects that exist in the same scope. Shared state can lead to very common bugs such as the race condition, a bug that happens when two concurrent functions which mutate the same variable or object cause undesirable effects.
  • Immutability: Prevents lossy data flow and the bugs that are associated with mutable data.

Syntax

  • Type "scala" into your terminal to experiment with Scala syntax, which is generally similar to Java
  • Defining variable types is not necessary in Scala, but there are two main types:
    • Val: Immutable, and Scala automatically creates a get function
    • Var: Mutable, and Scala creates both a getter and setter for the variable
  • Sample "hello world" function:

        object HelloWorld{
            def main(args: Array[String]) {
                 println("Hello World!")
            }
        }

Classes

  • Allows for multiple constructors(as long as they have different parameters) and default values for fields
  • The default constructor can essentially be contained in the first line of the class, because when you include inputs for a class scala will automatically make a constructor and getter or setters if applicable
  • Mixins: the user can extend Scala traits with a class or abstract class
  • Case Classes:
    • Immutable
    • Don't require the "new" keyword, because case classes have an apply method that takes care of object creation
    • Compares by structure rather than reference
    • Pattern matching: using the keyword match, you are able to write methods that take in a variable and figure out what kind of case class it is.
